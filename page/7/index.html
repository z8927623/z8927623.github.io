创建私有Pod（from车牛）

发表于 2015-07-03   |  

##Private Pod

参考文档：使用Cocoapods创建私有podspec CocoaPods Guides

###建立私有Spec Repo

车牛Private Spec Repo已经托管在我们自己的gitlab上。可以直接使用cheniu_pod，调用下面的命令即可：

pod repo add cheniu_pod http://git.souche.com/geliang/cheniu_pod.git

在~/.cocoapods/repos目录下可以看到cheniu_pod目录。

ps:由于使用的是http方式添加的cheniu_pod，所以添加的时候需要输入用户名和密码。

###创建Pod项目

这里只讲从0开始创建Pod项目，对已经存在的项目Pod化，根据文档自己研究下。创建Pod项目直接在对应的的目录下调用一下命令：

pod lib create cheniuTestPod

然后根据提示回答几个简单的设置问题就可以。前缀选择CNN，测试框架随便，反正也不会去用。创建完成之后可以在cheniuTestPod/Pod/Classes 和 Assets下面添加对应的类或者资源文件。在项目的Pods/DevelopmentPods/...中添加进来即可。

框架写好之后，可以在项目的cheniuTestPod/Example for cheniuTestPod/...中写一下简单的测试代码。

测试通过之后，在gitlab上面创建自己的repo并把自己的代码提交到远程库中。必须创建tag

git add -A
git commit -m 'xxxxxx'
git remote add origin xxxxx
git tag -m 'xxxxx' 0.1.0
git push -u origin master --tags

修改podspec文件，将多余的注释和字段删除。如果你的pod不需要资源文件一定要删掉s.resource_bundles字段，其他字段根据实际情况填写即可。 修改好之后可以做一次简单的验证pod lib lint,当看到

-> cheniuTestPod (0.1.0)
cheniuTestPod passed validation

时，说明验证是通过的。这里不验证其实也没关系，在你将这个podspec提交到私有Spec Repo中时也会做验证的

###向Spec Repo提交podspec

按照习惯提交之前要做两件事情

  • 删除podspec中的多余注释！
  • 做一次上面提到的验证工作。

然后使用下面的命令：

pod repo push cheniu_pod cheniuTestPod.podspec

完成之后我们可以到~/.cocoapods/repos/cheniu_pod/Specs中找到我们提交的spec，并且可以通过pod search查到我们提交的库。

###使用私有Pod

直接在我们自己的Podfile中加入

pod 'cheniuTestPod', '~> 0.1.0'

和source源

source 'http://git.souche.com/geliang/cheniu_pod.git'
source 'https://github.com/CocoaPods/Specs.git'

就可以了。加source非常重要，不然调用pod install时会找不到我们自己的私有库的。

###podspec升级，维护

维护podsepc，或者采用subspec新功能，参考文档自己摸索。
